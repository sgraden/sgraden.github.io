Workers Unbound

Workers Unbound is the next phase of Cloudflare's serverless platform, designed to execute complex, CPU-intensive workloads on the Edge without restrictive execution limits.

Role: Product Designer
Timeline: April 2020 - Current

Summary

Cloudflare Workers Unbound bridges the gap between lightweight edge scripts and heavy centralized cloud functions. Traditionally, developers had to choose between fast edge execution for simple routing and slow, containerized regions for heavy compute. Workers Unbound delivers infinite execution flexibility without CPU limits.

1

Remove CPU Execution Caps

Developers run CPU-heavy code (e.g. image processing, machine learning algorithms, database syncing) directly on the Edge.

arrow_downward
2

Leverage Chrome V8 Isolates

Rather than booting up heavy virtual machine containers with long cold starts, code runs in lightweight V8 Isolates that switch context in microseconds.

arrow_downward
3

Global Distribution Pipeline

Worker scripts are distributed to every single datacenter worldwide, allowing any Edge server to serve requests instantly.

arrow_downward
4

Drastic Infrastructure Savings

By eliminating container overhead, API gateway surcharges, and DNS request fees, Workers Unbound reduces serverless bills by up to 75% compared to AWS Lambda.

From a design and usability perspective, this removes complex cloud orchestration and region choices, giving developers a unified serverless solution. It matches simple, local-first developer experiences with global scalability, security, and industry-leading performance.
